# できること逆引き集（話しかけ方カタログ）

Claude Code に**日本語で話しかけるだけ**で富岳を操作できます。以下はタスク別の話しかけ方の例と、
裏で使われるツール・補足です。`「」`内がそのまま言える例です。

## 状態・自分の情報
| やりたいこと | 話しかけ方の例 | 裏のツール / 補足 |
|---|---|---|
| 富岳が動いているか | 「富岳の状態を見て」 | `cluster_status` |
| 自分のアカウント確認 | 「富岳での私のアカウント情報を見せて」 | `account_info`（取り違え防止）|
| 自分のディスク状況 | 「富岳でホームの使用量を見て」 | `run_command`（`du`/`quota`）|

## ジョブ
| やりたいこと | 話しかけ方の例 | 裏のツール / 補足 |
|---|---|---|
| 実行＋結果回収（推奨）| 「富岳で `hostname; lscpu` を実行して結果を見せて」 | `run_job`（投入→完了待ち→回収を一括）|
| ノード数・時間を指定 | 「2ノード・10分で〇〇を実行して」 | 資源上限ポリシー内で指定可 |
| グループを指定 | 「〇〇グループで投げて」 | 課金グループの指定 |
| 実行中ジョブ一覧 | 「いま動いてるジョブある？」 | `list_jobs` |
| 終わったジョブ | 「最近終わったジョブを見せて」 | `list_jobs(completed=true)`（直近24h）|
| 特定ジョブの状態 | 「ジョブ12345の状態は？」 | `job_status` |
| 結果を後から取得 | 「さっきの〇〇の結果を見せて」 | `fetch_result`（`<name>.result`）|
| ジョブ取消 | 「ジョブ12345を止めて」 | `cancel_job` |
| 長いジョブ | 「3時間のジョブを投げて。終わったら教えて」 | 待ち時間超過時はjobidが返る→後で状態確認 |

## ファイル
| やりたいこと | 話しかけ方の例 | 裏のツール / 補足 |
|---|---|---|
| 手元→富岳 | 「このスクリプトを富岳の `~/work/` に置いて」 | `stage_in`（既存は上書き）|
| 内容を直接作成 | 「`run.sh` という内容で富岳にファイルを作って」 | `stage_in`（content指定）|
| 富岳→手元/表示 | 「富岳の `~/work/out.log` を見せて」 | `stage_out` |
| 一覧 | 「富岳の `~/work/` の中身を見せて」 | `run_command`（`ls`）|

## コマンド（ログインノード上の軽作業）
| やりたいこと | 話しかけ方の例 | 裏のツール / 補足 |
|---|---|---|
| 確認・ビルド等 | 「富岳で `module avail` して」「`make` して」 | `run_command`（安全策ポリシー内）|

> 注意: `run_command` は**ログインノード上**の軽量コマンド用です。重い計算は必ずジョブ（`run_job`）で。
> 破壊的・危険なコマンドは安全策で拒否されます（[security.md](security.md)）。

## 組み合わせの例（エージェントが複数ステップを自動で）
- 「このコードを富岳に送って、ビルドして、1ノードで実行して、結果を見せて」
  → `stage_in` →（`run_command` でビルド）→ `run_job` → 結果回収、を一連で実行。
- 「失敗したジョブのログを見て、原因を直して再投入して」
  → `fetch_result`/`stage_out` で確認 → 修正 → `run_job` 再投入。

## 知っておくと便利
- **結果の置き場**: `run_job` の出力は富岳の `~/mcp-jobs/<name>.result` に集約されます。
- **完了の見え方**: 富岳では完了ジョブは実行キューから消え、最終状態は履歴(sacct, 直近24h)に出ます（ツールが自動判定）。
- **投入は少し時間がかかります**（富岳側で同期実行のため数十秒）。`run_job` が待ち受けます。
