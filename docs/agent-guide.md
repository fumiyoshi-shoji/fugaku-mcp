# 富岳エージェント運用ガイド（基本と困ったとき）

このファイルは `fugaku_help` ツールがエージェントへ返す知識ベース。富岳の基本操作と、よくあるエラーの対処をまとめる。

## 基本ワークフロー
- **計算は `run_job` が基本**: シェルコマンド（`commands`）を渡すと「投入→完了待ち→標準出力(stdout+stderr)を自動回収」まで一括。
- 状態確認: `cluster_status`（稼働状態）/ `list_jobs`（自分のジョブ）/ `account_info`（自分のアカウント・HOME・グループ）。
- ファイル: `stage_in`（送る）/ `stage_out`（取る）。**パスは絶対パス**。
- ログインノードの軽作業: `run_command`（ビルド・確認等）。**重い計算は必ずジョブ（`run_job`）で**。

## ジョブの投げ方（run_job）
- 例: `run_job(commands="hostname\ndate", name="test", nodes=1, elapse="00:10:00", rscgrp="small")`
- `name`: 英字始まり・英数と `. - _ @`・63文字以内。
- `nodes`: ノード数（上限あり）。1ノード = 48計算コア（A64FX）。
- `elapse`: 経過時間の上限 `HH:MM:SS`。
- `rscgrp`: リソースグループ（例 `small`）。課金グループ(`-g`)は `account_info` の `group` が自動で付く。
- 出力は `~/mcp-jobs/<name>.result`。`run_job` の戻り値 `result` に本文。後から `fetch_result(name)` でも取得可。
- 待ち時間（`wait_sec`）を超えた場合は `jobid` を返す → `job_status(jobid)` で確認。

## ジョブの状態の見方
- **投入(submit)はサーバ側で数十秒かかるのは正常**（同期実行のため）。
- 完了するとジョブは実行キューから消える。最終状態（`EXT`=正常終了、`CCL`=取消、`ERR`=エラー 等）は履歴に出る。`job_status` が自動で判定する。
- `list_jobs(completed=true)` は直近24時間の完了ジョブ。

## よくあるエラーと対処
- **ジョブが却下 / submit エラー**: `rscgrp`・`nodes`・`elapse`・課金グループを見直す。「rscgrp=small・1ノード・10分」のように明示。資源上限を超えていれば下げる。
- **submit がタイムアウトした**: 投入は成功していることがある。`list_jobs` で確認してから再投入する（**二重投入に注意**）。
- **ファイルが 409**: 既存ファイル。`stage_in` は上書きするので通常は問題なし。
- **Permission denied / IO Error（ファイル/コマンド）**: 自分の権限外（他ユーザ/システム領域）への操作。これは**正常な拒否**。自分の HOME 配下で操作する。
- **コマンドが安全策で拒否**: 危険なコマンド（`rm -rf` 等）はポリシーで拒否される。意図を変えるか、許可された範囲のコマンドを使う。
- **"No Jobs."**: ジョブが0件という**正常応答**（エラーではない）。

## パス・ファイルの注意
- ファイルAPIは**絶対パス**。`~` は展開されない（`run_command` の中では展開される）。
- ディレクトリ一覧は `run_command("ls -la <dir>")` で取得する（ファイルAPIの一覧は使わない）。

## MPI・並列（一般的な書き方／環境で要確認）
- 複数ノードのMPIは `run_job` の `commands` に Fujitsu TCS の作法で書く（例: `mpiexec -n <総プロセス数> ./a.out`）。ノード数は `nodes=` で指定。
- 正確なオプション（プロセス配置・スレッド数等）は富岳の公式ユーザガイド／`pjsub` リファレンスを参照（環境により異なるため、不確かなら決め打ちしない）。

## それでも分からないとき
- `run_command` で `module avail`・`pjacl`・`ls` 等を実行して環境・資源・ファイルを実地に調べる。
- 富岳の公式ユーザガイド／サポート（生成AIチャット AskDona）を参照する（将来、エージェントから直接問い合わせる連携を予定）。
