# FAQ・トラブルシューティング

## セットアップ・認証
**Q. VPNは必要ですか？**
不要です。富岳WebAPIはインターネット公開で、X.509証明書で認証します。

**Q. 証明書のパスフレーズを忘れました／証明書の有効期限が切れました。**
HPCI/R-CCSのポータルで証明書を再発行し、`./setup_user.sh 新しい証明書.p12` を再実行してください。

**Q. 複数のプロジェクトで使いたい。**
使うフォルダごとに `.mcp.json` を置けば、そのフォルダで起動したときに有効になります（[QUICKSTART](../QUICKSTART.md)）。

**Q. `python3 -m venv` や `pip install "mcp[cli]"` が失敗します。**
Python 3.10以上が必要です。3.14で失敗する場合は 3.12（`python3.12 -m venv .venv`）をお試しください。

## ツールが出てこない・反映されない
**Q. Claude Code に `fugaku` のツールが出てきません。**
1. **⌘Qで完全終了**してから再起動（ウィンドウの✕や⌘Wでは不十分）。
2. 使っているフォルダの直下に `.mcp.json` があるか確認。
3. 起動時の「`fugaku` を許可しますか？」を許可したか確認。

**Q. 設定やコードを変えたのに反映されません。**
- ツールの**増減**（新ツール追加など）→ **⌘Qで完全再起動**が必要。
- 環境変数の変更 → 同じく再起動で反映。

**Q. 自分の設定が正しいか確認したい。**
「富岳での私のアカウント情報を見せて」（`account_info`）で、使われているアカウント・HOME・グループが確認できます。

## ジョブ
**Q. ジョブが却下されます。**
リソースグループ／課金グループの指定が原因のことが多いです。「〇〇グループで」「rscgrp small で」「1ノード5分で」のように指定してください。資源上限（ノード数・時間）はポリシーで制限されている場合があります（[security.md](security.md)）。

**Q. 投入が遅い／数十秒かかります。**
正常です。富岳側でジョブ投入が同期実行されるため時間がかかります。`run_job` は完了まで待ち受けます。

**Q. ジョブの結果が見られません。**
- `run_job` を使えば自動で結果を回収します。手動の場合は「`~/mcp-jobs/<名前>.result` を見せて」。
- 完了ジョブは実行キューから消え、履歴（sacct）は**直近24時間のみ**。古い結果はジョブ出力ファイルを `stage_out` で取得してください。

**Q. 長いジョブで待ち時間を超えました。**
`run_job` は待ち時間超過時に jobid を返します。「ジョブ〇〇の状態は？」（`job_status`）で確認し、終わったら「結果を見せて」（`fetch_result`）。

## ファイル
**Q. ファイルのパスはどう指定する？**
**絶対パス**で指定してください。ファイル操作APIでは `~`（チルダ）は展開されません（コマンド実行では展開されます）。

**Q. 既存ファイルを上書きできますか？**
`stage_in` は既存ファイルを自動で上書きします。

## 安全策・権限
**Q. あるコマンドが拒否されました。**
安全策ポリシーにより危険なコマンドは拒否されます。許可範囲は `FUGAKU_CMD_MODE`（denylist/allowlist）で制御します（[security.md](security.md)）。

**Q. 他人のファイルやジョブは見えますか？**
いいえ。操作は**あなたのアカウント権限に閉じます**（富岳のOSレベルで分離）。システム/他者領域への読み書きは拒否されます。

## プライバシー・データの流れ（重要）
**Q. ファイルの中身やジョブ結果はどこへ行きますか？**
あなたのPC ↔ 富岳WebAPI 間はHTTPS（証明書認証）です。ただし、**AIアシスタントが読み取った内容（ファイル本文・ジョブ出力など）は、応答生成のためAIモデル（クラウド）に送られます**。機微データを扱う場合は、不要に `stage_out` や `cat` で中身をエージェントに渡さないでください。

**Q. 計算ノードからインターネットに出られますか？**
出られません。AI（頭脳）はローカルで動き、富岳側は計算の実行に専念します。両者は証明書付きHTTPSのAPI呼び出しでつながります（[README](../README.md) アーキテクチャ参照）。

## 履歴・運用
**Q. 利用履歴を残したい。**
ローカルは `FUGAKU_AUDIT_LOG`、富岳上への集約は `FUGAKU_AUDIT_REMOTE` を設定します。集計は `tests/audit_report.py`（[multi-user.md](multi-user.md)）。

## それでも解決しないとき
- `./setup_user.sh <証明書>.p12` を再実行して疎通（認証・ネットワーク）を切り分け。
- 「富岳の状態を見て」が通るか（=接続OK）から確認。
- 不明なエラーメッセージは管理者／開発者に共有してください。
